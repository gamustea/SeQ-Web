## Entry Point


Docker compose has two profiles:

## Database

## Config System

Two sources, merged via lazy loading in `API/src/modules/system/config_reading.py` (imported as `CR`):
1. `API/SecOpsConfig.json` â€” JSON config: DB credentials (fallback), prompts, color palettes, directory paths, module-specific settings.
2. `API/.env` â€” environment variables override JSON config. Required for OAuth (JWT_SECRET_KEY, JWT_ALGORITHM, ACCESS_TOKEN_EXPIRY_MINUTES, REFRESH_TOKEN_EXPIRY_DAYS) and database connection (POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB, etc.).

The root `.env` (at repo top level) is only for docker-compose (OpenVAS credentials). The API reads `API/.env` via python-dotenv.

## Auth

OAuth 2.0 with JWT. **JSON keys use camelCase** (`grantType`, `refresh_token`), not snake_case.

```
POST /oauth/token  {"grantType": "password", "username": "root", "password": "admin"}
```

All protected endpoints require `Authorization: Bearer <access_token>`.
